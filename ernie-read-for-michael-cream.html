01

The model, and who owns the data

Code red
Raised
I asked which models they run and pressed that the Anthropic API is closed, so you do not own the data and training comes with hoops.
Ernie said
Anthropic, version four point eight, not retraining, just setting expectations on the export. The live demo ran a Sonnet model, not the one named.
A strong answer
Name where the model and data live, separate building on a closed model from running on one, and say who owns the data.
My read
He named a model and dodged ownership and training. The gap is data ownership, training rights, and cost, not an absolute block, since closed APIs sign HIPAA BAAs.
Advisement
Own and fine-tune Qwen as the core, served via Ollama, so TLI owns the data. Keep PHI on the owned core, and run an omni reasoning layer over closed models only for non-PHI work or under a signed BAA.

01

Guardrails, hard versus soft, and life and death

Caution
Raised
I drew the line between hard guardrails, the deterministic rules baked in, and soft guardrails, trust earned over time, and asked if an owned model was on the roadmap.
Ernie said
They put guardrails on the model and set expectations, and triage cannot go live because the model cannot be trusted. Vince added it still hallucinates.
A strong answer
Separate hard from soft guardrails and show a path to earning trust over time.
My read
The premise that AI can never be trusted with care is wrong. Hard guardrails are deterministic, soft ones are earned by training. Ruling it out caps the product.
Advisement
Keep the deterministic hard guardrails and earn soft guardrail trust by training an owned model. Even then clinical triage stays Software as a Medical Device with human oversight, not an autonomous call.

03

Security versus compliance, and the go live plan

Caution
Raised
